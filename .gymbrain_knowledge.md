# GymBrain Knowledge Base: Lessons Learned & Faults Log
This file is a living document. The AI agent must ALWAYS consult and UPDATE this file after resolving implementation and debugging faults to prevent regressions. It must NOT be restructured from scratch, only modified and appended to.

## Backend / API
- **IVaultService Dependency**: `IVaultService` must be explicitly imported when used in Handlers. Missing using statements (e.g., `using GymBrain.Domain.Interfaces;`) cause silent build failures during `dotnet build`.
- **Entity Inheritance**: Ensure entities like `WorkoutSession` inherit from `BaseEntity`. Do not redefine properties like `CreatedAtUtc` that already exist in the base class.
- **Save Workout Endpoint Mapping**: When adding new functionality (like `SaveWorkoutCommand`), verify the endpoint is configured inside `WorkoutEndpoints.cs`.

## Frontend / React
- **Port Mismatches (401 Error)**: The API base URL configuration dictates where the frontend sends requests. If `VITE_API_URL` is missing in `client/.env`, it may fall back to `localhost:5000` while the `.NET API` runs on `localhost:5157`. This results in `401 Unauthorized` empty responses from conflicting background services. Fix by setting `VITE_API_URL=http://localhost:5157` in `client/.env`.
- **TypeScript Errors in Validation**: Ensure inline conditionals in TSX use valid syntax (e.g., chaining ternaries rather than logical OR `||` statements for placeholder strings) to avoid `npm run build` failure.
- **State Initialization Race Conditions**: Direct `localStorage.getItem` evaluation and short `setTimeout` buffers ensure proper visual hand-offs during sequential UI steps like Onboarding.

## General Pitfalls
- NEVER use PowerShell to modify `.csproj` file encoding (caused corruption previously).
- ALWAYS run `SafetyGate` on LLM JSON output before returning it to the client.
- ALWAYS seed exercises (empty tables cause orchestrator crashes).

## Audit Remediation (March 5, 2026)
- **Stale Test Assertions**: When refactoring prompts in `SystemPromptFactory`, always update the corresponding anti-hallucination test in `SystemPromptFactoryTests.cs` to match the new wording.
- **GUID Substring Collisions in Tests**: Exercise ID GUIDs contain common numeric substrings (e.g., `"100"` appears in `"10000001-..."`). Never use `NotContain("100")` as an assertion — use exact property matching like `Contain("\"reps\": 30")` instead.
- **Debug Artifacts**: Never commit `build_err.txt`, `error.txt`, or `out.json` to git. These are now gitignored.
- **Exercise Equipment Tags**: The `Exercise` entity now has an `Equipment` property (barbell, dumbbell, cable, machine, bodyweight) — use it when filtering exercises per user's available equipment.
